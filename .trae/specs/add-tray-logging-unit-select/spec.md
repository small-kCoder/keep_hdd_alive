# 系统托盘 + 活动日志 + 单位选择 Spec

## Why
当前 GUI 关闭即退出，用户无法后台运行；硬盘休眠/唤醒事件缺乏日志记录，无法评估间隔设置是否合理；时间单位固定为分钟，不够灵活。

## What Changes
- 新增系统托盘功能，关闭窗口时可选最小化到托盘或完全退出
- 新增硬盘休眠/唤醒事件日志，记录持续时间等关键信息用于分析
- 新增时间单位选择器（秒/分钟/小时），UI 数值自动转换

## Impact
- Affected specs: 无（新功能）
- Affected code: `自动(带gui)/keep_hdd_alive_gui.py`、`自动(带gui)/keep_hdd_alive.py`

## ADDED Requirements

### Requirement: 硬盘活动状态变更日志
当硬盘从活跃模式切换到空闲模式（即无外部活动、停止写入保活文件）时，系统 SHALL 自动记录一条详细日志，包含：
- 当前时间戳
- 本次活跃阶段持续了多久（从上次切换到活跃的时间到当前切换的时间）
- 本次活跃阶段内的保活写入次数
- 当前的间隔设置参数

当硬盘从空闲模式恢复到活跃模式时，系统 SHALL 记录一条日志，包含：
- 当前时间戳
- 空闲阶段持续了多久

#### Scenario: 硬盘进入空闲
- **WHEN** 检查周期内未检测到外部磁盘 I/O 活动
- **THEN** 日志输出"硬盘已空闲 | 活跃时长: 1h 30m | 保活写入: 18 次 | 间隔设置: 保活=30min 检查=30min"

#### Scenario: 硬盘恢复活跃
- **WHEN** 空闲态下检测到外部磁盘 I/O 活动
- **THEN** 日志输出"硬盘恢复活跃 | 空闲时长: 45m"

### Requirement: 系统托盘与后台运行
系统 SHALL 提供系统托盘图标，关闭窗口时弹出选项对话框。

- 托盘图标须在程序启动时创建，始终显示
- 关闭窗口（×）时弹出对话框，包含三个选项：
  - "最小化到系统托盘"：隐藏主窗口，程序继续在后台运行
  - "完全退出"：停止监控并退出程序
  - "取消"：不做任何操作
- 鼠标左键单击托盘图标：显示/隐藏主窗口
- 鼠标右键单击托盘图标：弹出菜单，包含"显示主窗口"和"退出"两项
- 托盘图标须有工具提示文本，显示当前状态（如"活跃模式"或"空闲模式"）

#### Scenario: 最小化到托盘
- **WHEN** 用户点击 × 按钮并选择"最小化到系统托盘"
- **THEN** 主窗口隐藏，托盘图标保持显示，监控继续运行

#### Scenario: 从托盘恢复窗口
- **WHEN** 用户单击托盘图标或右键菜单选择"显示主窗口"
- **THEN** 主窗口恢复显示

#### Scenario: 从托盘完全退出
- **WHEN** 用户右键托盘图标并选择"退出"
- **THEN** 监控停止，窗口关闭，程序退出

### Requirement: 时间单位选择
配置面板中 SHALL 增加一个时间单位下拉选择器，选项为"秒"、"分钟"、"小时"。

- 默认单位为"分钟"
- 切换单位时，保活写入间隔和活动检查间隔的数值自动转换（保持实际时长不变）
- 两个 SpinBox 的 suffix 和 range 根据所选单位动态调整
- 内部存储始终为秒，仅 UI 层做转换

#### Scenario: 切换单位
- **WHEN** 用户将单位从"分钟"切换为"秒"
- **THEN** 保活写入间隔从 30（分钟）变为 1800（秒），活动检查间隔同理

#### Scenario: 单位切换后 SpinBox 范围
- **WHEN** 单位选择"秒"，SpinBox 范围为 10~18000
- **WHEN** 单位选择"分钟"，SpinBox 范围为 1~300
- **WHEN** 单位选择"小时"，SpinBox 范围为 1~5
